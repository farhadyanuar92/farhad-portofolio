## Overview

Medistra Hospital had no patient-facing digital channel. Appointments were phone-based, queue management was manual, and patients had no visibility into their own journey through the hospital. This project built Medistra's first integrated patient engagement platform — a mobile application connected directly to the hospital's SIMRS infrastructure — giving patients digital access to bookings, medical resumes, real-time journey tracking, and notifications.

The platform also included a web-based CMS for hospital operations staff to manage doctors, schedules, promotions, and published hospital content.

## Problem

Hospital operations depended entirely on in-person and phone-based workflows. There was no digital touchpoint for patients, no queue visibility, and no way for the hospital to communicate proactively with patients. The SIMRS system held all clinical and operational data but had no API surface for external integrations — every integration had to be designed from scratch against a system with no external documentation.

## My Role

Project lead and full-stack engineer. Responsible for project coordination, feature planning, integration architecture, mentoring developers, and direct implementation of core booking, notification, and CMS systems. Also responsible for ensuring technical alignment with operational hospital workflows — the system had to work with how the hospital already operated, not force a process change.

## Key Challenges

**Integrating with an undocumented SIMRS system.** The hospital's core clinical system had no public API. Building the integration required deep analysis of existing data flows, direct collaboration with the SIMRS team, and designing transaction codes and synchronization points that didn't disrupt live clinical operations.

**Queue synchronization across systems.** Appointment booking had to synchronize in real time with the hospital's queue management service. A booking made in the app had to produce a valid, correctly sequenced queue entry in the operational system — with no duplicates and no missed slots.

**QR-based patient journey tracking.** Patients needed to track their own journey from check-in through to appointment completion. Implementing QR-based handoffs across check-in, waiting, consultation, and discharge required state consistency across multiple operational touchpoints.

## Technical Highlights

- Built SIMRS integration flows using **unique transaction codes** for appointment synchronization, ensuring correct ordering and preventing duplicate queue entries across concurrent bookings
- Implemented **mobile push notification infrastructure** for operational patient updates — appointment reminders, queue position changes, and post-visit communications
- Developed a full **CMS system** covering doctor profiles, scheduling, promotion management, and hospital information — designed for operational staff without engineering backgrounds
- Designed a **QR-based patient journey system** tracking patients across check-in, queue, consultation, and discharge with state consistency enforced at each transition
- Coordinated multi-system integrations — SIMRS, queue management, notification services, and CMS — while maintaining operational reliability throughout a live hospital environment

## Engineering Decisions

**Transaction code–based synchronization over direct database integration.** Rather than attempting a direct write into the SIMRS database (which carried risk of corrupting live clinical data), the integration was built on transaction code contracts — agreed-upon message formats that the SIMRS system already understood, reducing integration risk significantly.

**CMS designed for operational staff.** The content management system was scoped to match the actual operational needs of hospital staff — scheduling updates, doctor profile maintenance, promotion publishing — with workflows that matched how the team already thought about the content, not how a generic CMS would model it.

## Outcome

Patients gained digital access to appointment booking, real-time queue tracking, and medical history for the first time. The hospital reduced phone-based booking load and gained a direct communication channel to patients. The CMS gave operations staff full content control without engineering dependency. The platform became the hospital's primary digital patient touchpoint from launch.
